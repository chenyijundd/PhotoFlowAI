# PhotoFlow AI — AI 摄影选片工具

## 产品作战文档（MVP 正式版 · 已交付）

---

## 1. 产品定位

**产品名称**：PhotoFlow AI

**产品使命**：让摄影师把时间花在创作，而不是机械劳动。

**产品定位**：面向中国婚礼摄影工作室与职业摄影师的 **本地 AI 智能选片工具**。

帮助摄影师：
- 自动筛除废片
- 快速完成初选
- 降低人工挑片时间
- 提升交付效率

**差异化定位**：专为婚礼摄影工作流优化的本地 AI 选片工具。

**当前阶段目标**：不是做"AI 修图软件"，而是成为摄影工作流中的第一入口工具。

```
拍摄 → 选片（本产品切入点）→ 修图 → 交付
```

---

## 2. 目标用户

### 第一阶段核心用户

1. **婚礼摄影工作室** — 单次拍摄数量巨大，后期筛图耗时严重，有稳定商业需求，愿意为效率付费
2. **独立婚礼摄影师** — 一人承担拍摄+后期，熬夜挑片严重
3. **儿童摄影工作室** — 连拍数量多，重复图多，闭眼废片多

### 明确不做的用户

V1 不面向：普通自拍用户、网红美颜用户、手机修图用户、设计师、Photoshop 重度用户

---

## 3. 用户核心痛点

```
摄影结束 → 导出几千张照片 → 人工挑片 → 删除模糊/闭眼/重复/构图差 → 进入修图
```

**核心问题**：
1. **挑片耗时巨大** — 单场 2000~5000 张，挑片 3~6 小时
2. **重复劳动严重** — 不断放大、对比、删除，工作疲劳
3. **判断不稳定** — 疲劳导致漏删、标准不统一
4. **新人难培训** — 工作室难以统一选片标准

**本软件目标**：10 分钟完成初筛，节省 80% 挑片时间。

### 关键架构模式

- **后台线程 + 轮询**：所有 AI 检测任务异步执行，前端通过 `setInterval` 轮询进度
- **页面接管模式**：LightboxPage / ComparePage 通过 Context 状态接管整个视口

- **Repository 模式**：数据访问层统一封装，支持自动迁移
- **模块解耦**：每个 AI 模块独立可运行（CLI + Service + API）
- **React.memo + 自定义比较器**：精确控制 ImageCard 重渲染

## 8. 用户操作流程（当前实现）

```
Step 1: 打开软件
    ↓
Step 2: 📁 导入照片目录 → 自动扫描(JPG/PNG/HEIC + RAW) + RAW预览提取 + 生成缩略图
    ↓
Step 3: 🤖 一键 AI 分析（确认对话框，按当前筛选范围执行）
    ├── Step 1/5: 闭眼检测（MediaPipe + EAR）→ 致命缺陷，自动判废
    ├── Step 2/5: 模糊检测（多区域拉普拉斯 V2）→ 仅标记，不自动筛
    ├── Step 3/5: 连拍分组（EXIF 时间聚类 ≤2s）
    ├── Step 4/5: 重复检测（dHash + 结构相似度）→ 仅查剩余照片
    ├── Step 5/5: 最佳推荐（多维度排名）
    └── 级联设计：每张照片只归入最先命中的分类
    ↓
Step 4: 筛选器浏览
    ├── 全部 — 查看所有照片
    ├── 待处理 — 逐张审核未决定照片
    ├── 已选 — 查看精选结果
    └── 废片 — 复查废片（防止误删）
    ↓
Step 5: ⚡ 一键选片（5 步：闭眼→废片，模糊→标记，连拍/重复→选最佳→废片其余，干净→加星）
```

## 10. AI 模糊检测 V2 核心算法

> 这是 V1 之后的重大升级，解决了背景虚化(bokeh)被误判为模糊的问题。
> **角色变更**：模糊检测已从"自动判废"降级为"仅标记提醒"，由摄影师人工判断。

**算法**：多区域拉普拉斯方差 + 中心加权

```
1. 将图片分为 4×4 = 16 个区域
2. 计算每个区域的拉普拉斯方差
3. 中心区域加权（人像通常位于中心）
4. 取 Top 50% 区域的中位数（忽略纯色背景区域）
5. 最终得分 = weighted × 0.4 + top_median × 0.6
6. 阈值 55：< 55 判定为模糊
```

**对比 V1**：V1 在 bokeh 人像（清晰主体 + 虚化背景）上误判率极高，V2 通过多区域分析正确识别清晰主体。

## 11. 连拍分组 + 最佳推荐

### 连拍分组（Burst Grouper）

- **方法**：EXIF 拍摄时间滑动窗口聚类
- **阈值**：2 秒间隔自动切分连拍组
- **最少成员**：2 张照片

### 最佳推荐（Best Selector）

- **排名维度**：模糊得分 → 文件体积(同分时) → 分辨率(再同分时)
- **算法**：排除废片 → 排除闭眼 → 排除模糊 → 按清晰度排序 → 前 10% 视为并列，用文件体积打破平局
- **级联优先级**：闭眼照片不参与任何最佳推荐竞争

### 级联检测管道设计

每张照片只出现在一个 AI 分类中，按问题严重程度分层：

```
闭眼 (L1: 致命) → 模糊 (L2: 质量) → 连拍 (L3: 组织) → 重复 (L4: 冗余)
```

- L1 闭眼：自动判废，跳过所有后续检测
- L2 模糊：仅标记提示，由摄影师人工判断
- L3 连拍：时间聚类分组，组内推荐最佳
- L4 重复：感知哈希查重，避免冗余交付

## 12. RAW 格式预览支持

> 婚礼摄影师 100% 拍摄 RAW 格式。RAW 文件不是标准图片格式，无法被 Pillow/OpenCV 直接读取。

**实现方式**：提取内嵌 JPEG，不碰 RAW 像素数据。

大多数相机在 RAW 文件内嵌入了一张全尺寸 JPEG 预览图（通常 2-8 MB）。导入时通过 `rawpy` 提取该 JPEG，保存到 `cache/raw_previews/` 目录。

```
.CR2 文件 → rawpy.extract_thumb() → 内嵌 JPEG → cache/raw_previews/{id}.jpg
                                                    ↓
                                     ┌──────────────┴──────────────┐
                                     ↓              ↓              ↓
                                缩略图生成      AI 模糊检测     AI 闭眼检测
                                (Pillow)       (OpenCV)       (MediaPipe)
```

**支持的 RAW 格式**（13 种）：

| 格式 | 相机品牌 |
|------|----------|
| .CR2 / .CR3 | Canon |
| .NEF / .NRW | Nikon |
| .ARW / .SRF | Sony |
| .DNG | Adobe / Leica / DJI / 手机 |
| .ORF | Olympus / OM System |
| .RAF | Fujifilm |
| .RW2 | Panasonic |
| .PEF | Pentax |
| .SRW | Samsung |

**关键设计决策**：
- **零 RAW 解码** — 仅提取内嵌 JPEG，不处理 Bayer 阵列数据
- **自动降级** — 若预览丢失或提取失败，自动回退到原始文件路径
- **原始路径保留** — DB 中 `file_path` 始终指向原始 RAW 文件，`raw_preview_path` 指向提取的 JPEG
- **`PhotoRecord.readable_path`** — 统一入口，所有图像读取代码使用此属性
