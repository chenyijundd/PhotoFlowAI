# 闭眼检测实施计划

> **决策结论**：模糊检测从「自动判废」降级为「建议复核」；闭眼检测作为新的主力自动判废信号。
>
> 制定日期：2026-05-31 | **实施完成**：2026-06-01
>
> ✅ Phase 1-5 全部完成。级联管道已实现：
> 闭眼(L1) → 模糊(L2) → 连拍(L3) → 重复(L4) → 最佳推荐

---

## 一、背景与动机

### 1.1 当前问题

| 问题 | 严重程度 | 说明 |
|------|----------|------|
| 模糊检测假阳性高 | 🔴 严重 | V2 多区域 Laplacian 仍将散景人像、暗光照片误判为模糊 |
| 「一键选片」盲目信任模糊检测 | 🔴 严重 | `_run_cull_all()` Step 1 无条件拒绝所有 `is_blur=1` 的照片 |
| 闭眼检测是空壳 | 🔴 严重 | `eye_detector.py` 永远返回 `eyes_open: True`，产品文档却标记为「已完成」 |
| 无人工审核环节 | 🟡 中等 | 被误判的照片只能事后在废片堆里手动翻找恢复 |

### 1.2 为什么闭眼检测比模糊检测更有价值

婚礼摄影场景分析：

- **专业摄影师拍出模糊照片的概率极低（<1%）** — 现代相机 AF 系统非常可靠
- **合影中有人闭眼/半闭眼的概率高（5-15%）** — 这是婚礼摄影的刚需痛点
- **闭眼照片 100% 不可修复** — 而轻微模糊可以 AI 修复或缩图使用
- **闭眼检测结果可肉眼验证** — 用户信任度高；模糊检测结果主观性强
- **EAR（眼睛纵横比）算法成熟可靠** — 基于面部 landmarks 的几何测量，假阳性率极低

### 1.3 模糊检测的新定位

不删除模糊检测，但改变其角色：

```
之前：模糊 → 自动判废（高风险）
之后：模糊 → 标记展示，由摄影师人工判断（低风险）
```

极端模糊（如慢门拖影）仍然会被标记，但不由机器做最终决定。

## 二、技术方案：MediaPipe + EAR

### 2.1 算法原理

**Eye Aspect Ratio (EAR)** — Soukupová & Čech, 2016：

```
         p1
    . . . . . .
 p2 .         . p3       EAR = (∥p2-p6∥ + ∥p3-p5∥) / (2 × ∥p1-p4∥)
    .         .
 p4 .         . p5
    . . . . . .
         p6

睁眼时 EAR ≈ 0.30~0.40
闭眼时 EAR ≈ 0.10~0.18
半闭眼时 EAR ≈ 0.18~0.25
```

### 2.2 技术选型

| 候选方案 | 优点 | 缺点 | 选择 |
|----------|------|------|------|
| **MediaPipe Face Mesh** | 精度高、轻量、Python 支持好、已在项目依赖中 | 需要下载模型文件 | ✅ **采用** |
| dlib + 68 landmarks | 经典方案 | 安装复杂、Windows 编译问题多 | ❌ |
| OpenCV Haar Cascade | 无额外依赖 | 精度低、易漏检 | ❌ |
| InsightFace / MTCNN | 精度极高 | 依赖 PyTorch/TensorFlow，太重 | ❌ |

**决策**：使用 MediaPipe Face Mesh，与产品文档架构设计一致，已列在技术选型中。

### 2.3 处理流程

```
输入图片
    ↓
MediaPipe Face Detection（检测所有人脸）
    ↓
对每张人脸 → Face Mesh → 提取 468 个 landmarks
    ↓
取眼部 6 个关键点（左右眼各 6 个）
    ↓
计算左右眼 EAR
    ↓
EAR < 0.18 → 闭眼
0.18 ≤ EAR < 0.22 → 半闭眼（也标记）
EAR ≥ 0.22 → 睁眼
    ↓
任意一只眼闭眼/半闭眼 → is_closed_eye = 1
多张人脸 → 只要有一人闭眼 → is_closed_eye = 1
无人脸检测到 → is_closed_eye = 0, face_detected = False
```

### 2.4 阈值设计

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `EAR_CLOSED_THRESHOLD` | 0.18 | 低于此值判定为闭眼 |
| `EAR_HALF_CLOSED_THRESHOLD` | 0.22 | 低于此值判定为半闭眼（也标记） |
| 最终判定 | EAR < 0.22 | 闭眼 + 半闭眼都标记，给摄影师决定权 |

#### Task 1.2：重写 `backend/ai/eye_detection/eye_detector.py`

**新实现**：

```python
"""
PhotoFlow AI - Eye Detection Module (Real Implementation)

Uses MediaPipe Face Mesh to extract eye landmarks, then computes
Eye Aspect Ratio (EAR) to detect closed / half-closed eyes.

Algorithm reference: Soukupová & Čech, 2016
"""

import math
import time
import cv2
import numpy as np
import mediapipe as mp

# MediaPipe Face Mesh indices for eye landmarks
# Left eye:  33, 160, 158, 133, 153, 144  (outer → inner → outer)
# Right eye: 362, 385, 387, 263, 373, 380
LEFT_EYE_IDX  = [33, 160, 158, 133, 153, 144]
RIGHT_EYE_IDX = [362, 385, 387, 263, 373, 380]

# Tunable thresholds
EAR_CLOSED_THRESHOLD = 0.18       # < this → closed
EAR_HALF_CLOSED_THRESHOLD = 0.22  # < this → half-closed (also flagged)

# MediaPipe singleton (lazy init, reused across calls)
_face_mesh = None

def _get_face_mesh():
    global _face_mesh
    if _face_mesh is None:
        _face_mesh = mp.solutions.face_mesh.FaceMesh(
            static_image_mode=True,
            max_num_faces=10,       # wedding group photos
            refine_landmarks=True,  # higher precision around eyes
            min_detection_confidence=0.5,
        )
    return _face_mesh


def _ear(landmarks, eye_indices, img_w, img_h):
    """Compute Eye Aspect Ratio for one eye."""
    pts = []
    for idx in eye_indices:
        lm = landmarks.landmark[idx]
        pts.append((lm.x * img_w, lm.y * img_h))
    
    # Vertical distances
    v1 = math.dist(pts[1], pts[5])  # p2-p6
    v2 = math.dist(pts[2], pts[4])  # p3-p5
    # Horizontal distance
    h  = math.dist(pts[0], pts[3])  # p1-p4
    
    if h < 1e-6:
        return 0.0
    return (v1 + v2) / (2.0 * h)


def detect_eyes(image_path: str) -> dict:
    """Analyze a single image for eye state.
    
    Returns:
        {
            "file": str,
            "eyes_open": bool,       # True if all detected eyes are open
            "score": float,          # lowest EAR across all eyes (0~1, lower = more closed)
            "face_detected": bool,
            "num_faces": int,
            "closed_count": int,     # number of faces with closed/half-closed eyes
            "per_face": [...]        # detailed per-face EAR values
        }
    """
    t0 = time.perf_counter()
    
    # Read image (Unicode-safe)
    data = np.fromfile(image_path, dtype=np.uint8)
    img = cv2.imdecode(data, cv2.IMREAD_COLOR)
    if img is None:
        raise FileNotFoundError(f"Cannot decode image: {image_path}")
    
    h, w = img.shape[:2]
    rgb = cv2.cvtColor(img, cv2.COLOR_BGR2RGB)
    
    # Run MediaPipe Face Mesh
    face_mesh = _get_face_mesh()
    results = face_mesh.process(rgb)
    
    if not results.multi_face_landmarks:
        elapsed = (time.perf_counter() - t0) * 1000.0
        return {
            "file": image_path,
            "eyes_open": True,
            "score": 1.0,
            "face_detected": False,
            "num_faces": 0,
            "closed_count": 0,
            "per_face": [],
            "processing_time_ms": elapsed,
        }
    
    # Evaluate each face
    all_ears = []  # lowest EAR per face
    closed_count = 0
    per_face = []
    
    for i, face_landmarks in enumerate(results.multi_face_landmarks):
        left_ear = _ear(face_landmarks, LEFT_EYE_IDX, w, h)
        right_ear = _ear(face_landmarks, RIGHT_EYE_IDX, w, h)
        min_ear = min(left_ear, right_ear)
        
        is_closed = min_ear < EAR_HALF_CLOSED_THRESHOLD
        if is_closed:
            closed_count += 1
        
        all_ears.append(min_ear)
        per_face.append({
            "face_index": i,
            "left_ear": round(left_ear, 4),
            "right_ear": round(right_ear, 4),
            "min_ear": round(min_ear, 4),
            "is_closed": is_closed,
        })
    
    # Overall: closed if ANY face has closed eyes
    overall_min_ear = min(all_ears) if all_ears else 1.0
    eyes_open = closed_count == 0
    
    elapsed = (time.perf_counter() - t0) * 1000.0
    
    return {
        "file": image_path,
        "eyes_open": eyes_open,
        "score": round(overall_min_ear, 4),
        "face_detected": True,
        "num_faces": len(results.multi_face_landmarks),
        "closed_count": closed_count,
        "per_face": per_face,
        "processing_time_ms": elapsed,
    }
```

**验收标准**：
- 输入睁眼人像 → `eyes_open: True, score > 0.22`
- 输入闭眼人像 → `eyes_open: False, score < 0.22`
- 输入无人物风景照 → `face_detected: False, eyes_open: True`
- 输入多人合影（有人闭眼）→ `closed_count >= 1, eyes_open: False`
- CLI 可独立运行：`python eye_detector.py --input ./test.jpg`

#### Task 2.4：修改「一键选片」逻辑（`_run_cull_all`）

**这是最关键的改动。**

**当前逻辑**：
```
Step 1: 模糊照片 → 自动拒绝
Step 2: 重复照片 → 自动拒绝  
Step 3: 连拍组   → 最佳加星，其余拒绝
```

**新逻辑**：
```
Step 1: 闭眼照片 → 自动拒绝（高置信度，不可修复）
Step 2: 重复照片 → 自动拒绝（不变）
Step 3: 连拍组   → 最佳加星，其余拒绝（不变）
```

模糊照片：**不再自动拒绝**，仅在缩略图上保留 `BLUR` 徽章提示，由摄影师自行判断。

## 五、数据流对比

### 之前（有问题）

```
AI 分析 4 步：模糊 → 重复 → 连拍 → 最佳
                    ↓
一键选片：模糊(自动拒绝) + 重复(自动拒绝) + 连拍(推荐)
                    ↓
            问题：好照片被模糊检测误杀
```

### 之后（修复后）

```
AI 分析 5 步：模糊(仅标记) → 闭眼 → 重复 → 连拍 → 最佳
                                ↓
一键选片：闭眼(自动拒绝) + 重复(自动拒绝) + 连拍(推荐)
  模糊：仅 BLUR 徽章提示，不自动拒绝
                                ↓
            摄影师在废片堆中只看真正有问题的照片
            模糊照片由摄影师自己判断
```

## 六、风险与缓解

| 风险 | 概率 | 缓解措施 |
|------|------|----------|
| MediaPipe 在 Windows 上安装/运行问题 | 中 | 提前验证；提供 fallback 到 OpenCV Haar 的降级方案 |
| 侧脸/遮挡导致人脸漏检 | 中 | 设置 `min_detection_confidence=0.5`，对未检测到人脸的照片标记 `face_detected=False` |
| 戴眼镜导致 EAR 计算偏差 | 低 | MediaPipe 的 iris landmarks 对眼镜鲁棒性较好；半闭眼阈值 0.22 留有余量 |
| 闭眼检测假阳性（眯眼笑被误判） | 低 | 半闭眼阈值 0.22 对此类情况较宽容；眯眼笑通常 EAR > 0.22 |
| 处理速度慢（批量大时） | 中 | Face Mesh 每张约 50-100ms，2000 张约 2-3 分钟，可接受 |
