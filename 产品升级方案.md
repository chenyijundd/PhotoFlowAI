### 一、问题分析

当前所有 AI 检测阈值都是**硬编码**的模块级常量。不同摄影师、不同拍摄场景（户外明亮婚礼 vs 室内暗光晚宴）对"废片"的定义差异很大，一套固定阈值无法适配所有场景。

**核心痛点：**

| 痛点 | 具体表现 |
|------|---------|
| 阈值不可调 | 摄影师觉得闭眼漏检太多，但不敢改代码 |
| 场景不匹配 | 户外强光下的人像眯眼判断 vs 室内暗光的闭眼判断完全不同 |
| 无反馈闭环 | 改了阈值后不知道效果变好还是变差 |
| 一次性配置 | 每次分析用同一套参数，不能按项目灵活切换 |

### 二、当前所有硬编码阈值总览

#### 2.1 闭眼检测 (`backend/ai/eye_detection/eye_detector.py`)

| 参数 | 当前值 | 含义 | 调高效果 |
|------|--------|------|---------|
| `BLINK_SCORE_THRESHOLD` | `0.55` | MediaPipe 眨眼 blendshape 分数阈值 | 更严格，更多照片被标闭眼 |
| `EAR_CLOSED_THRESHOLD` | `0.18` | 眼睛纵横比 — 低于此值判为闭合（无 blendshape 回退方案） | 更严格，更多照片被标闭眼 |
| `EAR_HALF_CLOSED_THRESHOLD` | `0.20` | AND 逻辑中的半闭眼候选线（需 blendshape 交叉确认） | 更多照片进入闭眼候选 |

**判定逻辑：** `EAR < 0.20` **且** `blink_score > 0.55` → 标的为闭眼（AND 逻辑）

#### 2.2 模糊检测 (`backend/ai/blur_detector_v2/detector.py`)

| 参数 | 当前值 | 含义 | 调高效果 |
|------|--------|------|---------|
| `BLUR_THRESHOLD` | `25.0` | 最终综合分数低于此值 → 模糊 | 更严格，更多照片标模糊 |
| `PREVIEW_SHARP_THRESHOLD` | `80.0` | 800px 预览分数高于此值 → 跳过全尺寸分析（明确清晰） | 更多照片免检，速度快但可能漏判 |
| `PREVIEW_BLUR_THRESHOLD` | `25.0` | 800px 预览分数低于此值 → 跳过全尺寸分析（明确模糊） | 更多照片直接判模糊 |

**判定逻辑：** 三级流水线 — 800px 预览快速筛 → 边缘得分进入 16 区域全尺寸分析 → 加权合成最终分（中心加权 0.4 + 中位数 0.6）

#### 2.3 重复检测 (`backend/ai/duplicate_detector/`)

| 参数 | 当前值 | 含义 | 调高效果 |
|------|--------|------|---------|
| `HAMMING_PREFILTER` | `15` | dHash 汉明距离上限（超过此值不进入 SSIM 精确比对） | 更严格匹配，更多候选对 |
| `SSIM_THRESHOLD` | `0.60` | 结构相似度最低分（低于此值判定为重复） | 更严格，需要更高相似度才算重复 |
| `TIME_WINDOW_GAP` | `300` | EXIF 时间间隔窗口（秒），超过 5 分钟的间隙拆分为独立时间桶 | 更大的时间窗口，可能合并更多照片 |

**判定逻辑：** 时间窗口预分组 → dHash 多索引哈希表粗筛 → SSIM 精确判定

#### 2.4 连拍分组 (`backend/ai/burst_grouper/grouper.py`)

| 参数 | 当前值 | 含义 | 调高效果 |
|------|--------|------|---------|
| `BURST_GAP_SECONDS` | `2.0` | 相邻照片时间间隔 ≤ 此值归为同一连拍组 | 更宽松，更多照片被归组 |
| `MIN_BURST_SIZE` | `2` | 最少照片数，少于此数不构成连拍组 | —（不建议调） |

**判定逻辑：** 按 EXIF 时间排序 → 相邻照片间隔 ≤ 2s 归为一组 → 丢弃成员 < 2 的组

#### 2.5 最佳推荐 (`backend/ai/best_selector/selector.py`)

| 参数 | 当前值 | 含义 | 调高效果 |
|------|--------|------|---------|
| `BLUR_TIE_PCT` | `0.10` | 模糊分数差异小于 10% → 视为平局，进入下一轮比较 | 更多照片进入尺寸/分辨率决选 |
| `SIZE_TIE_PCT` | `0.20` | 文件大小差异小于 20% → 视为平局，进入分辨率决选 | 更少的照片被直接淘汰 |

**判定逻辑：** 排除已拒标/模糊/闭眼 → 模糊分排序 → 10% 平局 → 文件大小排序 → 20% 平局 → 分辨率决选

### 三、方案设计

#### 3.1 三档预设

提供三档预设，一键切换全部阈值：

```
┌─────────────┬──────────────────────┬──────────────────────┐
│             │  严格 (Strict) │  宽松 (Lenient)    │
│             │  漏检少 · 误检多      │  漏检多 · 误检少      │
├─────────────┼──────────────────────┼──────────────────────┤
│ 适合场景     │  重要场合（仪式、合影） │  快速过片（准备环节）   │
│             │  不想错过任何好照片     │  不想被假阳性淹没      │
│ 闭眼        │  更容易标闭眼          │  更难标闭眼            │
│ 模糊        │  更容易标模糊          │  更难标模糊            │
│ 重复        │  更容易标重复          │  更难标重复            │
│ 连拍        │  更容易归组            │  更难归组              │
│ 最佳        │  更宽容（多张平局）     │  更严苛（快刀斩乱麻）   │
└─────────────┴──────────────────────┴──────────────────────┘
```

中间档 = **标准 (Standard)** = 当前硬编码值。

#### 3.2 各档具体阈值

##### 闭眼检测

| 参数 | 严格 | 标准 | 宽松 |
|------|------|------|------|
| `BLINK_SCORE_THRESHOLD` | **0.45** | 0.55 | **0.70** |
| `EAR_CLOSED_THRESHOLD` | **0.22** | 0.18 | **0.15** |
| `EAR_HALF_CLOSED_THRESHOLD` | **0.24** | 0.20 | **0.17** |

- 严格：blink 分只要 0.45 就及格 + EAR 半闭眼线放宽到 0.24 → 更多照片满足 AND 条件 → 更多闭眼被检出
- 宽松：blink 分必须 ≥ 0.70 + EAR 半闭眼线收紧到 0.17 → 只有明确闭眼才被检出

##### 模糊检测

| 参数 | 严格 | 标准 | 宽松 |
|------|------|------|------|
| `BLUR_THRESHOLD` | **35.0** | 25.0 | **18.0** |
| `PREVIEW_SHARP_THRESHOLD` | **60.0** | 80.0 | **100.0** |
| `PREVIEW_BLUR_THRESHOLD` | **35.0** | 25.0 | **18.0** |

- 严格：最终分 ≤ 35 才判模糊 → 更多照片被判模糊 → 宁可误杀不可放过
- 宽松：最终分 ≤ 18 才判模糊 → 只有明确模糊才标 → 疑罪从无
- 严格模式下预览免检门槛降低（60 就跳过，当前 80），加快分析速度

##### 重复检测

| 参数 | 严格 | 标准 | 宽松 |
|------|------|------|------|
| `HAMMING_PREFILTER` | **20** | 15 | **10** |
| `SSIM_THRESHOLD` | **0.50** | 0.60 | **0.70** |
| `TIME_WINDOW_GAP` | **600** | 300 | **120** |

- 严格：dHash 汉明距离 ≤ 20 + SSIM ≥ 0.50 + 10 分钟窗口 → 大幅放宽 → 更多重复
- 宽松：dHash 汉明距离 ≤ 10 + SSIM ≥ 0.70 + 2 分钟窗口 → 极严 → 只有几乎完全相同的照片才判重复

##### 连拍分组

| 参数 | 严格 | 标准 | 宽松 |
|------|------|------|------|
| `BURST_GAP_SECONDS` | **3.0** | 2.0 | **1.5** |
| `MIN_BURST_SIZE` | 2 | 2 | 2 |

- 严格：3 秒间隔容差 → 更多照片被归入连拍组 → 组内对比更全面
- 宽松：1.5 秒间隔 → 只有高速连拍的片段才归组 → 减少需对比的照片组

##### 最佳推荐

| 参数 | 严格 | 标准 | 宽松 |
|------|------|------|------|
| `BLUR_TIE_PCT` | **0.20** | 0.10 | **0.05** |
| `SIZE_TIE_PCT` | **0.30** | 0.20 | **0.10** |

- 严格：20% 模糊平局 + 30% 大小平局 → 进入决选的照片更多 → 但最终标「最佳」的也更多、更分散
- 宽松：5% 模糊平局 + 10% 大小平局 → 每步淘汰更狠 → 只有绝对突出的照片才是最佳

#### 4.4 预设数据结构

```python
# backend/config/presets.py

@dataclass
class PresetThresholds:
    """All tunable thresholds for a single sensitivity preset."""
    # Eye detection
    blink_score_threshold: float
    ear_closed_threshold: float
    ear_half_closed_threshold: float
    # Blur detection
    blur_threshold: float
    preview_sharp_threshold: float
    preview_blur_threshold: float
    # Duplicate detection
    hamming_prefilter: int
    ssim_threshold: float
    time_window_gap: float
    # Burst grouping
    burst_gap_seconds: float
    min_burst_size: int
    # Best selector
    blur_tie_pct: float
    size_tie_pct: float

@dataclass
class Preset:
    id: str           # "strict" | "standard" | "lenient"
    name: str         # "严格" | "标准" | "宽松"
    description: str  # 适用场景说明
    thresholds: PresetThresholds
```

#### 4.5 API 设计

```
GET  /api/config/presets              → 返回三档预设列表（含阈值详情）
GET  /api/config/presets/active       → 获取当前项目激活的预设
PUT  /api/config/presets/active       → 设置当前项目激活的预设
Body: {"preset_id": "strict"}
```

#### 4.6 存储方式

**按项目存储激活预设 ID**，存入每个项目的 SQLite 数据库：

```sql
-- 在项目数据库中新增一个 kv 配置表
CREATE TABLE IF NOT EXISTS config (
    key   TEXT PRIMARY KEY,
    value TEXT NOT NULL
);

-- 激活预设
INSERT OR REPLACE INTO config (key, value) VALUES ('active_preset', 'standard');
```

项目创建时默认 `standard`。

### 五、边界场景与交互规则

#### 5.1 切换预设后

| 状态 | 行为 |
|------|------|
| 未分析 | 下次分析使用新预设（无需额外操作） |
| 已分析、未选片 | 提示「重新分析以应用新预设」，一键分析按钮高亮 |
| 已分析、已选片 | 提示「重新分析会覆盖当前选片结果」⚠️ |
| 正在分析中 | 预设下拉菜单灰色禁用，hover 提示「分析进行中」 |

#### 5.2 一键选片

一键选片使用分析时的预设阈值（不独立设置），确保分析和选片的一致性。

#### 5.3 单模块分析

如果用户只运行了模糊检测（而非完整 5 步分析），后续再运行闭眼检测时，两者使用**同一套**预设。切换预设后需要重新运行各模块。

#### 5.4 API 参数覆盖优先

各 API 端点已有的阈值参数（如 `BlurDetectRequest.threshold`）优先级高于预设。即：显式传参 > 预设值 > 硬编码默认值。

#### 5.5 预设影响范围总结

| 操作 | 是否受预设影响 |
|------|:---:|
| 一键 AI 分析（5 步级联） | ✅ |
| 一键选片（5 步级联） | ✅ |
| 单独运行闭眼检测 `/api/ai/eye-detect` | ✅ |
| 单独运行模糊检测 `/api/ai/blur-detect` | ✅ |
| 单独运行重复检测 `/api/ai/duplicate-detect` | ✅ |
| 单独运行连拍分组 `/api/ai/burst-group` | ✅ |
| 恢复手动操作的历史状态 | ❌（不受预设影响） |
| 回收站操作 | ❌（不受预设影响） |
| 导出 | ❌（不受预设影响） |

### 六、实施步骤

- [ ] `backend/config/presets.py` — 定义三档预设 + `get_preset()` / `list_presets()` 函数

- [ ] 各检测器 `detect_*()` 方法增加可选 `thresholds: PresetThresholds` 参数

- [ ] `ai_service.py` 分析流程读取预设并传入各检测器

#### Phase 4: 验证（0.5h）

- [ ] 切换预设 → 运行分析 → 确认阈值生效
