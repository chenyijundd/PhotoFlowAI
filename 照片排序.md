# PhotoFlow AI — 照片排序方案

> 版本: v1.0 · 2026-05-31

---

## 一、设计原则

1. **每个 Tab 有其语义** — 排序策略与该 Tab 的用途深度绑定
2. **摄影师手动操作 > AI 自动操作** — 用户手动加星/废片的照片，在对应 Tab 中排在最前面
3. **时间线是基线** — 没有特殊排序需求的 Tab，按 `created_time` (拍摄时间) 排序
4. **数据库持久化** — 排序依据通过数据库字段存储，重启后排序不变

---

## 二、可用排序字段

| 字段 | 含义 | 来源 | 适用场景 |
|------|------|------|---------|
| `created_time` | 照片拍摄时间 (EXIF) | 导入时从 EXIF 读取 | 基本时间线 |
| `blur_score` | 模糊程度 (越高越模糊) | AI 模糊检测 | 模糊 Tab |
| `duplicate_group` | 重复照片分组 ID | AI 重复检测 | 重复 Tab |
| `burst_group` / `burst_position` | 连拍组 ID / 组内序号 | AI 连拍分组 | 连拍 Tab |
| `manually_operated_at` | 用户最近一次手动操作时间 | 手动加星/废片时记录 | 已选/废片 Tab 置顶 |
| `star_rating` | 加星状态 (0/1) | 用户操作 | — |
| `updated_at` | 任意最近更新时间 (含 AI) | 自动记录 | — |

## 三、各 Tab 排序规则

### 3.1 主 Tab (顶部筛选栏)

| Tab | 一级排序 | 二级排序 | 说明 |
|-----|---------|---------|------|
| **全部** | `created_time ASC` | — | 时间线回溯，故事脉络 |
| **已选** ⭐ | `manually_operated_at DESC NULLS LAST` | `created_time ASC` | 手动操作的排最前面，其余按时间线 |
| **待处理** | `created_time ASC` | — | 从旧到新逐一过片，不遗漏 |
| **废片** 🗑️ | `manually_operated_at DESC NULLS LAST` | `created_time ASC` | 手动标记的废片排最前面以确认，其余按时间线 |

**已选/废片特殊规则：**

- AI 一键选片自动加星或废片的照片 → `manually_operated_at = NULL`
  - 排列在普通已选/废片之后，按 `created_time` 排序
- 摄影师手动按 Space/X 或点击星标操作的照片 → `manually_operated_at = now()`
  - 排列在该 Tab 的最前面，按操作时间从新到旧
- 在同一张照片上再次手动操作 → 更新时间戳，保持置顶

#### 示例场景

1. 摄影师导入 500 张婚礼照片
2. 点击「一键选片」，AI 自动处理：
   - 50 张模糊 → 废片 (`manually_operated_at = NULL`)
   - 30 张重复 → 废片 (`manually_operated_at = NULL`)
   - 20 张连拍最佳 → 加星 (`manually_operated_at = NULL`)
3. 摄影师浏览「已选」Tab：看到 20 张 AI 推荐的连拍最佳，按拍摄时间线排列
4. 摄影师手动按 Space 额外加星 5 张 → 这 5 张跳到「已选」Tab 最前面
5. 摄影师切到「废片」Tab：80 张 AI 废片按时间线排列，方便按拍摄场景确认
6. 摄影师发现某张被误标废片 → 按 X 取消废片 → 该照片从废片 Tab 消失

### 3.2 AI 分类子 Tab (分析后出现)

| Tab | 一级排序 | 二级排序 | 说明 |
|-----|---------|---------|------|
| **模糊** | `blur_score DESC` | `created_time ASC` | 最模糊的优先处理 |
| **重复** | `duplicate_group ASC` | `file_name ASC` | 同组相邻方便对比 |
| **连拍** | `burst_group ASC` | `burst_position ASC` | 同组相邻还原帧序 |
| **最佳推荐** | `created_time ASC` | — | 精选时间线 |

## 五、技术方案详解

### 5.1 manually_operated_at

```
ALTER TABLE photos ADD COLUMN manually_operated_at TEXT DEFAULT NULL;
```

- 类型 `TEXT` (ISO 8601 时间戳)，默认 `NULL`
- 仅在**摄影师手动**操作时设置，AI/批量操作不触及
- `NULL` = 非手动操作，排序时排在后面

### 5.2 Repository 方法签名变更

```python
def update_star_rating(self, image_id: str, star_rating: int, is_manual: bool = False) -> bool
def update_reject_status(self, image_id: str, is_rejected: int, is_manual: bool = False) -> bool
```

- `is_manual=True` → 额外设置 `manually_operated_at = now()`
- `is_manual=False` (默认) → 不设置 `manually_operated_at`

### 5.3 排序 SQL

```sql
-- 已选 Tab
ORDER BY manually_operated_at DESC NULLS LAST, created_time ASC

-- 废片 Tab
ORDER BY manually_operated_at DESC NULLS LAST, created_time ASC

-- 全部 / 待处理 Tab
ORDER BY created_time ASC

-- 模糊 Tab
ORDER BY blur_score DESC, created_time ASC

-- 重复 Tab
ORDER BY duplicate_group ASC, file_name ASC

-- 连拍 Tab
ORDER BY burst_group ASC, burst_position ASC

-- 最佳推荐 Tab
ORDER BY created_time ASC
```

### 5.4 调用链路

**手动操作链路：**
```
前 端 (Space/X 按键) → photoApi.ts → IPC → main.cjs → HTTP PATCH
  → detail_service.py → repo.update_star_rating(id, 1, is_manual=True)
  → SQL: UPDATE photos SET star_rating=1, manually_operated_at='2026-...', updated_at='...'
```

**AI 自动操作链路：**
```
一键选片 → photo_service.py _run_cull_all
  → repo.update_star_rating(id, 1)           ← is_manual=False (默认)
  → repo.update_reject_status(id, 1)          ← is_manual=False (默认)
  → 不设置 manually_operated_at (保持 NULL)
```

**批量用户操作链路 (burst filmstrip)：**
```
用户点击「接受最佳」→ burst_service.py accept_best_in_burst
  → repo.update_star_rating(id, 1, is_manual=True)
  → repo.update_reject_status(id, 1, is_manual=True)
```
